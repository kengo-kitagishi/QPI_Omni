# シンプルな固定範囲正規化への変更

## ユーザーの状況分析

- **画像内**: 外れ値が存在
- **画像間**: ばらつきは小さい
- **目的**: アライメント精度向上

## 現在の問題

```python
def to_uint8(img):
    img_min = np.min(img)  # 画像ごとに異なる
    img_max = np.max(img)  # 画像ごとに異なる
    normalized = (img - img_min) / (img_max - img_min)
    return (normalized * 255).astype(np.uint8)
```

**問題点**:
- 外れ値1つで全体のスケールが変わる
- 画像間で対応が取れない
- 同じピクセル値が異なるuint8値にマッピング

## 255ギリギリまで使うメリットは？

実は**ほとんどない**：

1. **ECCアルゴリズムの特性**
   - 勾配ベースの最適化
   - 絶対値よりも**相対的な変化**を見る
   - 0-255でも50-200でも結果は同じ

2. **むしろデメリット**
   - 外れ値の影響を受けやすい
   - オーバーフロー/アンダーフローのリスク
   - ノイズの影響が大きい

3. **中間範囲のメリット**
   - 外れ値に強い
   - 安定した変換
   - ノイズの影響が小さい

## 推奨実装: シンプル固定範囲版

```python
def to_uint8(img, vmin=-1.5, vmax=2.0):
    """
    固定範囲でuint8に変換（アライメント用）
    
    Parameters:
    -----------
    vmin, vmax : float
        クリッピング範囲（位相画像のrad値）
    
    Note:
    -----
    - 全画像で一貫した変換を保証
    - 外れ値の影響を軽減
    - 19_gaussian_backsub.pyの範囲(-1.1~1.5)に基づく
    """
    # クリッピング（外れ値除去）
    clipped = np.clip(img, vmin, vmax)
    
    # 0-255に正規化
    normalized = (clipped - vmin) / (vmax - vmin)
    
    return (normalized * 255).astype(np.uint8)
```

### 範囲の選択

位相画像の典型的な範囲（19_gaussian_backsub.pyより）:
- ヒストグラム範囲: -1.1 ~ 1.5 rad
- 安全マージン込み: **-1.5 ~ 2.0 rad** (推奨)

この範囲なら：
- ほとんどの正常ピクセルをカバー
- 外れ値は自動的にクリッピング
- 画像間で一貫した変換

## より柔軟な実装（オプション）

範囲をカスタマイズしたい場合：

```python
def to_uint8(img, vmin=-1.5, vmax=2.0, use_data_range=False):
    """
    uint8に変換
    
    Parameters:
    -----------
    use_data_range : bool
        True: 画像のmin-maxを使用（現在の動作）
        False: 固定範囲を使用（推奨）
    """
    if use_data_range:
        vmin = np.min(img)
        vmax = np.max(img)
    
    clipped = np.clip(img, vmin, vmax)
    normalized = (clipped - vmin) / (vmax - vmin)
    return (normalized * 255).astype(np.uint8)
```

## 期待される効果

1. **一貫性**: 全画像で同じスケール
2. **外れ値耐性**: 外れ値が自動的にクリッピング
3. **精度向上**: 画像間の対応が正確に
4. **安定性**: ノイズの影響が小さい
