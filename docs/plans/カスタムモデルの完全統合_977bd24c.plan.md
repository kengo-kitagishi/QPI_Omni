# カスタムOmniposeモデルの完全統合プラン

## 前提条件の整合性確認結果

### Omniseggerが期待する形式

1. **フォルダ構造**:
   ```javascript
         dataset/
         ├── xy1/
         │   ├── phase/
         │   │   ├── t001xy1c1.tif
         │   │   ├── t002xy1c1.tif
         │   │   └── ...
         │   ├── masks/       ← Omniposeが生成
         │   │   ├── t001xy1c1_cp_masks.png  (PNG形式)
         │   │   └── または t001xy1cp_masks.tif  (TIF形式)
         │   ├── seg/         ← Omniseggerが生成
         │   └── cell/        ← Omniseggerが生成
   ```




2. **ファイル命名規則**: `basename_t[frame]xy[position]c[channel].tif`

- c1 = phase/bright field（必須）
- c2, c3... = fluorescence（オプション）

3. **マスク形式**:

- ラベル画像（0=背景、1,2,3...=各細胞）
- PNG (uint8/uint16) または TIF
- **flowは不要**（`--no_npy`使用のため）

### 現在の不一致点

- ユーザーの画像ファイル名が異なる
- 推論スクリプトの出力が `_masks.tif` + `_binary.tif`
- 出力フォルダが `inference_out/` ではなく `masks/` である必要

## 実装内容

### 1. 画像準備用Pythonスクリプト作成

**新規ファイル**: `omnisegger/prepare_images_for_omnisegger.py`任意の画像フォルダをOmnisegger形式に変換:

- ファイル名を `t001xy1c1.tif` 形式に変換
- `xy1/phase/` フォルダ構造を作成
- 蛍光画像がある場合は対応

### 2. カスタムモデル推論スクリプト作成

**新規ファイル**: `omnisegger/run_custom_omnipose.py`Omnisegger互換の推論を実行:

```python
# 主な機能:
- カスタムモデルでバッチ推論
- nchan=1, nclasses=3 に対応
- Omnisegger期待形式で出力:
    * masks/ フォルダに保存
    * filename + c1_cp_masks.png 命名
    * 0=背景、1,2,3...=各細胞のラベル画像
- GPU/CPU切り替え
- パラメータ調整（diameter, flow_threshold, mask_threshold）
```



### 3. MATLAB側の修正

[`omnisegger/SuperSegger-master/settings/loadConstants.m`](omnisegger/SuperSegger-master/settings/loadConstants.m) と [`omnisegger/SuperSegger-master/batch/BatchSuperSeggerOpti.m`](omnisegger/SuperSegger-master/batch/BatchSuperSeggerOpti.m) を修正:

- Omniposeモデルパスとパラメータを設定可能に
- `genOmniposeCommand` を動的に生成
- カスタムコマンド（自作Python）も指定可能

### 4. 統合ワークフロースクリプト作成

**新規ファイル**: `omnisegger/custom_model_workflow.py`完全自動化:

```python
# ステップ1: 画像をOmnisegger形式に準備
prepare_images(input_dir, output_dir)

# ステップ2: カスタムモデルで推論
run_custom_inference(model_path, image_dir, params)

# ステップ3: MATLAB処理の準備完了通知
print("Ready for MATLAB tracking and analysis")
```

## 技術的詳細

### マスク出力仕様

- **形式**: PNG (uint16推奨) または TIF
- **ピクセル値**: 0=背景、1,2,3...=各細胞のID
- **ファイル名**: 元画像 `t001xy1c1.tif` → マスク `t001xy1c1_cp_masks.png`
- **保存先**: `xy1/masks/` フォルダ

### 対応するOmnisegger機能

セグメンテーション後、以下のMATLAB機能が使用可能:

- 細胞トラッキング（時系列リンク）
- 蛍光解析（fluorescence quantification）
- カイモグラフ生成
- 細胞系譜解析
- SuperSeggerViewer による可視化
- Clist出力（細胞データ表）

## メリット

- カスタムモデルでセグメンテーション
- Omniseggerの全機能（追跡・解析・可視化）を活用
- 既存データの再処理も可能
