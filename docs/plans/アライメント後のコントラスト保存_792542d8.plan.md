# アライメント後のコントラスト保存修正

## 問題の詳細

### 現在の処理フロー

```python
# 216行: uint8に変換（固定範囲 -5.0 ~ 2.0）
target_uint8 = to_uint8(target_img)  # -5.0~2.0 → 0~255

# 230-233行: uint8でwarpAffine
aligned_uint8 = cv2.warpAffine(target_uint8, warp_matrix, (w, h), ...)

# 234-235行: float64に戻す（元画像のmin-max使用）← ここが問題！
aligned_img = aligned_uint8.astype(np.float64) / 255.0  # 0~255 → 0~1
aligned_img = aligned_img * (np.max(target_img) - np.min(target_img)) + np.min(target_img)
```

### 問題点

**変換時と復元時で異なる範囲を使用**:
- 変換: -5.0 ~ 2.0（固定）
- 復元: min(target_img) ~ max(target_img)（可変）

**結果**: コントラストが変わる

## 解決策

### オプション1: 固定範囲で復元（推奨）

uint8への変換と同じ固定範囲を使って復元：

```python
# 変換適用
h, w = target_img.shape
aligned_uint8 = cv2.warpAffine(
    target_uint8, warp_matrix, (w, h),
    flags=cv2.INTER_LINEAR + cv2.WARP_INVERSE_MAP
)

# 固定範囲で復元
vmin, vmax = -5.0, 2.0
aligned_img = aligned_uint8.astype(np.float64) / 255.0  # 0~1
aligned_img = aligned_img * (vmax - vmin) + vmin  # -5.0~2.0
```

**メリット**:
- コントラストが完全に保存される
- 全画像で一貫した処理
- シンプル

### オプション2: 元画像に直接適用

uint8を経由せず、元のfloat64画像に直接warpAffine：

```python
# 元画像に直接適用
h, w = target_img.shape
aligned_img = cv2.warpAffine(
    target_img.astype(np.float32), warp_matrix, (w, h),
    flags=cv2.INTER_LINEAR + cv2.WARP_INVERSE_MAP
).astype(np.float64)
```

**メリット**:
- uint8変換が不要
- 元の値が完全に保存される
- より正確

**デメリット**:
- ECCの計算はuint8で行われるので、変換マトリックスはuint8用
- float64に適用すると微妙な誤差が出る可能性

### オプション3: ハイブリッド

ECCはuint8で計算し、変換は元画像に適用：

```python
# ECC計算（uint8）
target_uint8 = to_uint8(target_img)
correlation, warp_matrix = cv2.findTransformECC(
    alignment_reference_uint8, target_uint8, warp_matrix, ...
)

# 変換は元画像に適用（float64）
aligned_img = cv2.warpAffine(
    target_img.astype(np.float32), warp_matrix, (w, h),
    flags=cv2.INTER_LINEAR + cv2.WARP_INVERSE_MAP
).astype(np.float64)
```
